# World Cup 2026 Live Tracker

A live tracker for the 2026 FIFA World Cup. Real scores, standings, schedule, stats,
team profiles, a live-now banner, per-match detail (lineups + timeline), a projected
knockout bracket, and an **AI daily digest** — all updating automatically from public
data. No manual entry; this is a tracker, not a simulator.

Single-file React + Vite app (`src/App.tsx`) plus one Netlify function for the AI digest.

## Run

```bash
npm install
npm run dev            # http://localhost:5173 (scores/stats work; the AI digest needs netlify dev — see below)
npm run build          # static build into dist/
```

## Data

- **Scores / standings / stats / live games / lineups** come from ESPN's public,
  keyless scoreboard + summary endpoints (CORS-open), fetched directly in the browser.
- Data loads on launch and **polls every 30s while a match window is active**.
- Group fixtures (matchups, dates, venues) are baked into `src/App.tsx`.
- Knockout is a **projection** from current standings until real KO games begin
  (June 28); it then shows real results.

## AI daily digest (Gemini, via a Netlify function)

The digest at the top is generated by Gemini from the current tournament state. The
API key is **never** in the client — it lives in a serverless function.

1. In Netlify: **Site settings → Environment variables → add `GEMINI_API_KEY`**.
2. Deploy. The browser calls `/.netlify/functions/digest`; the function calls Gemini.
3. Cost control: the digest is generated **once per day per browser** (cached in
   `localStorage`), with a manual refresh button.

The model only narrates the facts it's given (today's results, upcoming games, live
games, standings) — it never invents scores. Model is set in `netlify/functions/digest.mjs`.

### Local testing of the digest

```bash
npm i -g netlify-cli      # if needed
echo "GEMINI_API_KEY=your_key_here" > .env   # gitignored — do NOT commit
netlify dev               # serves the app AND the function locally
```

Under plain `npm run dev` the function isn't available, so the digest panel simply
stays hidden (everything else works).

## Deploy (Netlify)

- Base directory: `worldcup-2026-standalone`
- Build: `npm run build` · Publish: `dist` · Functions: `netlify/functions` · Node 22
  (all preconfigured in `netlify.toml`)
- Set `GEMINI_API_KEY` in env vars for the digest.
